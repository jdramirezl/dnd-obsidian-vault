# Saving Throws

## Original Text

A saving throw—also called a save—represents an attempt to resist a spell, a trap, a poison, a disease, or a similar threat. You don't normally decide to make a saving throw; you are forced to make one because your character or monster is at risk of harm.

To make a saving throw, roll a d20 and add the appropriate ability modifier. For example, you use your Dexterity modifier for a Dexterity saving throw.

A saving throw can be modified by a situational bonus or penalty and can be affected by advantage and disadvantage, as determined by the DM.

Each class gives proficiency in at least two saving throws. The wizard, for example, is proficient in Intelligence saves. As with skill proficiencies, proficiency in a saving throw lets a character add his or her proficiency bonus to saving throws made using a particular ability score. Some monsters have saving throw proficiencies as well.

The Difficulty Class for a saving throw is determined by the effect that causes it. For example, the DC for a saving throw allowed by a spell is determined by the caster's spellcasting ability and proficiency bonus.

The result of a successful or failed saving throw is also detailed in the effect that allows the save. Usually, a successful save means that a creature suffers no harm, or reduced harm, from an effect.


## Condensed Version

**Saving Throws**: Reactive rolls to resist harmful effects — spells, traps, poisons, diseases, and similar threats.

### Basic Mechanic

1. An effect forces you to make a save (you don't choose to)
2. Roll **d20** + ability modifier (+ [[03-Proficiency-Bonus|proficiency bonus]] if proficient)
3. Compare to the **DC** set by the effect
4. **Meet or beat DC** = success (no harm or reduced harm)

### Proficiency in Saving Throws

- Each class grants proficiency in **at least two** saving throw types
- Proficiency = add **proficiency bonus** to that save
- Some monsters also have saving throw proficiencies

| Class | Proficient Saves |
|-------|-----------------|
| Cleric | Wisdom, Charisma |
| Fighter | Strength, Constitution |
| Rogue | Dexterity, Intelligence |
| Wizard | Intelligence, Wisdom |

> [!tip] DM Tip
> Saving throws are reactive — the creature doesn't choose to make one. Call for a save when something *happens to* a creature (spell, trap, poison). Call for an ability check when a creature *tries to do* something.

### How the DC Is Determined

- **Spells**: 8 + caster's spellcasting ability modifier + caster's proficiency bonus
- **Traps, poisons, environmental effects**: Set by the DM or the effect's description
- **Monster abilities**: Listed in the monster's stat block

### Modifiers

- **Situational bonuses/penalties** may apply (DM's call)
- **Advantage/Disadvantage** can affect saving throws
- **Proficiency bonus** added if proficient in that save type

### Success and Failure

- **Success**: Usually no harm, or reduced harm (e.g., half damage)
- **Failure**: Full effect applies (damage, condition, etc.)
- The specific result is described by the effect that triggers the save

**Key Concept**: Saving throws are forced reactions to danger, not voluntary actions. You roll d20 + ability modifier (+ proficiency if applicable) against a DC set by the threat. Each class is proficient in two saves, making those saves significantly more reliable.
