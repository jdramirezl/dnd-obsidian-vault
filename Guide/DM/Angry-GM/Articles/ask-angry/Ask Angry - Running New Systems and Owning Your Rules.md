# Ask Angry: Running New Systems and Owning Your Rules

> **Source**: [Original Article](https://theangrygm.com/ask-angry-running-new-systems-and-owning-your-rules/) | **Category**: Ask Angry | **Date**: February 2, 2016

---

## Key Advice

- **When learning a new system, don't start a campaign.** Run a limited adventure — a few sessions at most — using pregenerated characters and a prewritten adventure. Starter sets and quick starts exist for this purpose.
- **Every RPG system has hidden design intentions** baked into its mechanics that encourage a certain style of play. D&D thinks in encounters and adventuring days; Savage Worlds thinks in quick moment-to-moment actions; GURPS makes you hate algebra; Fate sucks.
- **Give the game a chance to tell YOU how to run it.** Don't force your favorite system's mindset onto a new game. Listen to what the mechanics are trying to do.
- **The GM owns the rules. Players don't apply rules — they declare actions.** The GM resolves those actions using the rules as one tool among many. Players can raise concerns, but the GM makes the ruling and moves on.
- **Consistency is the GM's obligation.** The rules must work the same way every time. If Strength-based Intimidation works this week, it works next week too.

## Summary — Part 1: Running New Systems

When running a brand-new RPG system, treat it like being a brand-new GM. Run a disposable one-shot or short adventure first — ideally a published quick-start — so you can learn the mechanics without campaign stakes. This also reveals the game's hidden design philosophy: the themes, tone, and play patterns the designers intended.

Beyond mechanics, immerse yourself in the genre and setting if it's unfamiliar. Read the game's recommended source material, watch related shows, or binge the media that inspired the setting. Angry ran a Marvel Heroic RPG campaign by binge-watching the X-Men cartoon and running side stories set between episodes — because that was the one corner of Marvel he actually liked.

The core lesson: running games is how you get good at running games, but you have to listen to the game first before you can run it well.

## Summary — Part 2: Owning Your Rules

Players sometimes cherry-pick between "realism" and "rules as written" depending on which benefits them — invoking physics when it helps and citing the rulebook when physics doesn't. This isn't malicious; players naturally use every tool to win. But the premise is flawed: **the rules are not the players' tools. The rules belong to the GM.**

The RPG loop works like this: GM describes → players declare actions → GM resolves using rules. Players describe what they do; the GM decides which rules apply and how. If a player gets set on fire and there's no extinguishing rule, the GM creates one based on how fire works. If an enemy gets set on fire next week, the same ruling applies.

The GM must never get into rules debates during play. A player can raise a concern once. The GM makes a ruling, explains briefly, and moves on. The approach to a task determines which rules apply — a barbarian looming over someone and flexing uses Strength + Intimidation; a halfling getting in someone's face with a death stare uses Charisma + Intimidation. The player describes the action; the GM picks the mechanic.

## Actionable Steps

1. **For new systems:** Run a throwaway one-shot with pregens and a published adventure before committing to a campaign.
2. **Absorb the genre** if it's unfamiliar — read, watch, or play media that matches the setting's tone and themes.
3. **Don't import your old system's mindset.** Let the new game's mechanics teach you what kind of play it supports.
4. **Establish early that the GM adjudicates rules.** Players declare actions; the GM decides which mechanics apply based on the approach.
5. **Be consistent.** Once you make a ruling, it becomes precedent. Write it down if needed.
6. **Shut down rules lawyering after one exchange.** Player raises concern → GM explains ruling → game moves on. No debates.
7. **Match the mechanic to the approach, not the intention.** What the player is *doing* determines the ability score and skill, not what they *want to happen*.

> [!tip] Bottom Line
> Listen to the game before you try to run it. And once you're running it, own the rules — because if the GM doesn't control how the rules work, nobody does.
