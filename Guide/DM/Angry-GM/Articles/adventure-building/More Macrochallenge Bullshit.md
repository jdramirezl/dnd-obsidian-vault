# Oh, No! More Macrochallenge Bullshit

## Key Advice

D&D is not a complete game — it's a **toolkit** for building games. When you write an adventure, you are doing **real game design**. The difference between a functioning game and a *good* game is the **Macrochallenge**: the way all the individual encounters add up to test players beyond any single obstacle. D&D's built-in default Macrochallenge is **Resource Management**, and it deserves respect because it automatically produces well-designed gameplay if you follow the adventure-building instructions.

## Summary

Angry revisits and clarifies his earlier, poorly explained concept of "Macrochallenges." The core argument unfolds in layers:

**D&D is not a game — it's a game-building toolkit:**
- A game needs goals, rules, and obstacles. D&D as written has no goal — it relies on adventures and GMs to create goals.
- Without a defined goal, there can be no defined obstacles. D&D provides pre-made game elements (monsters, traps, spells) from which obstacles are built.
- Writing an adventure *is* game design. You're not following instructions to build a Lego castle — you're designing a unique game.

**Micro- and Macrochallenge defined:**
- A **Microchallenge** is a single encounter — one test the players must overcome. It's the equivalent of a single turn in a board game.
- A **Macrochallenge** is how all the encounters add up to challenge players beyond any individual obstacle. It's what makes the game "greater than the sum of its parts."
- These are relative terms, not absolutes. A scene can be a Macrochallenge relative to its encounters, and a Microchallenge relative to the whole adventure.

**Every D&D scenario has a Macrochallenge — the question is whether it's a good one:**
- A scenario that's just "win every encounter" has a bland, shitty Macrochallenge. Players never think beyond the obstacle in front of them.
- A good Macrochallenge forces players to consider each encounter in the context of the larger goal.

**There is no list of Macrochallenges to pick from:**
- Every Macrochallenge is a uniquely designed play experience. It *is* the game.
- Terms like "dungeon crawl," "mystery," or "intrigue" are vague summaries, not design templates.
- Designing a good Macrochallenge *is* designing a good scenario. They're the same thing.

**D&D's default Macrochallenge is Resource Management:**
- If you follow D&D's adventure-building instructions (multi-encounter adventuring days, proper rest pacing), the game automatically produces a resource-management challenge.
- Players must manage spell slots, hit points, hit dice, potions, ammunition, and other expendable resources across multiple encounters.
- The penalty for failure (retreating to rest) is about how it *feels* — slinking home, wasting table time on camp logistics.
- D&D 3E's designers built this into the engine so that GMs who follow the instructions get a well-designed game without needing game design expertise.

**This was a 3E innovation:**
- Earlier editions had attrition gameplay, but lacked a rigorous toolkit that consistently delivered balanced resource-management challenges.
- Encounter balance was "optional" in AD&D 2E. There were no defined rates of attrition for GMs to design around.
- 3E (and 4E) empowered GMs to build good, stable, reliable gameplay experiences from the toolkit.

## Actionable Steps

1. **Recognize that writing adventures is game design** — you're not just slapping encounters together, you're designing a unique game experience.
2. **Every scenario needs a Macrochallenge** — ask yourself: "What must the players do, and to do so, what must they manage/overcome beyond individual encounters?"
3. **Don't look for a list of Macrochallenges** — there isn't one. Design your scenario's challenge creatively, the same way a board game designer would.
4. **Respect D&D's default Resource Management Macrochallenge** — if you follow the multi-encounter adventuring day structure, you get a well-designed game for free.
5. **Don't break the default without a replacement** — if you run single-encounter adventuring days or let players rest freely, you've removed the default Macrochallenge and need to design something to replace it.
6. **If you want to replace the default, prepare to do real game design** — there's no shortcut. You're designing a game, and that takes creativity, careful design, and practice.

> [!tip] The Toolkit Principle
> "D&D is not a complete game: it's a pile of Lego pieces from which one can build a complete game." When you write an adventure, you are a game designer. The quality of your game depends on whether you design a Macrochallenge that makes the whole greater than the sum of its encounters — or just build an obstacle course.
