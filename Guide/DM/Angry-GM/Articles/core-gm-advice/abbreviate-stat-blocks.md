# I Can't Abbreviate This Article But I Can Abbreviate a Stat Block

**Source:** [The Angry GM](https://theangrygm.com/abbreviate-stat-blocks/)
**Date:** September 23, 2015
**Category:** Core GM Advice

---

## Summary

An analysis of user interface design in RPGs — specifically D&D 5E — and a practical demonstration of how to abbreviate monster stat blocks so they can be included inline in adventure text. Angry argues that WotC has never seriously considered how their products are actually used at the table, resulting in a terrible user experience for both players and GMs. He traces the history of stat block presentation from AD&D 2E through 3E, 4E, and 5E, then provides a method for condensing 5E stat blocks to roughly one-third their original size.

---

## Key Points

### The UI Problem in D&D

- **RPGs have a user interface** — character sheets, stat blocks, book layout, page numbers, indexes — and D&D's is terrible.
- **The character sheet** fails to include critical information: weapon damage types, weapon keywords (Versatile, Loading), ranges, and especially spell details (range, components, concentration, effects).
- **The PHB/DMG/MM** are poorly organized for reference use: small page numbers, no chapter signifiers on pages, inconsistent alphabetical ordering in the MM, and a chaotic DMG layout.
- **Published adventures** (e.g., Out of the Abyss) reference monsters by name only, forcing GMs to flip through the Monster Manual mid-session with no legal way to have multiple stat blocks visible simultaneously.

### Historical Approaches to Stat Blocks

| Edition | Approach | Strength | Weakness |
|---------|----------|----------|----------|
| **AD&D 2E** | Abbreviated inline stat blocks in adventures; full entries in Monster Manual for worldbuilding | Clean separation of "run at table" vs. "build adventures" | Limited by simpler system |
| **D&D 3E/3.5** | Started with abbreviated blocks, moved to Tactical Encounter Format | Tactical format was excellent UI | Monster complexity made abbreviation impractical; tactical format had page count/layout costs |
| **D&D 4E** | Stat blocks designed around UI from the start; monsters built to suit their presentation | Best GM-facing interface | Blocks still large; multiple monsters per encounter ate space; felt "soulless" to some |
| **D&D 5E** | Identical stat blocks in MM and adventures; trained users to accept MM page references | — | Abandoned all previous UI progress; passed the problem to GMs |

### How to Abbreviate a 5E Stat Block

**What to cut:**
- Alignment (for unaligned creatures)
- HP dice code (keep only the number)
- Full ability scores (keep only modifiers, e.g., `S-1 D+2 C+1 I-2 W+0 Ch-1`)
- Passive Perception (calculable from Wisdom modifier)
- Languages (unless unusual)
- Challenge Rating (keep XP for awards)
- Default assumptions in attacks: "Melee Weapon Attack," "reach 5 ft.," "one target" — only note when different

**What to keep:**
- AC, HP, Speed
- Ability modifiers
- Skills, senses, damage resistances/immunities
- Traits (abbreviated to reminder-level text)
- Attacks with to-hit, damage, damage type, and any special effects

**Result:** A 21-line kobold stat block becomes 8 lines. A kobold + giant lizard fits in 13 lines. Ten monsters can fit on a single reference page.

### The Bigger Point

> There's more to making an RPG than designing mechanics. The interface — how people use and play your game — should be as much a part of the design as any other aspect. D&D is a good game with a bad user experience, and the community has been trained to accept that.
